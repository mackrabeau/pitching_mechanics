PHASE 1: Analytics Pipeline  ✅ COMPLETE
──────────────────────────────────────────────

Step 1a ✅ — Data layer
  Files: pitchlens/data/poi_metrics.py
  - load_poi()      → 411-row biomechanics + metadata DataFrame (joined on session_pitch)
  - load_hp()       → 1098-row strength/athleticism DataFrame (filtered to pitchers)
  - load_combined() → both, ready for modeling
  - Column groups: KINEMATIC_COLS, KINETIC_COLS, ENERGY_COLS, GRF_COLS,
                   HP_STRENGTH_COLS, HP_ROM_COLS
  - Derived features: BMI, normalized elbow/shoulder moments
  - Fixed: metadata 'session' column collision resolved via drop-before-merge

Step 1b ✅ — Biomechanics velo model
  Files: pitchlens/analytics/velo_model.py
  - BiomechanicsVeloModel: HistGradientBoostingRegressor (handles NaN natively)
  - 78 features: kinematics + energy flow + GRF + anthropometrics + playing_level_enc
  - GroupKFold CV (5-fold, grouped by session — no leakage)
  - CV R²=0.550  RMSE=3.06 mph on held-out sessions
  - SHAP TreeExplainer for signed per-feature contributions
  - Top features: elbow_transfer_fp_br, thorax_distal_transfer_fp_br,
                  max_shoulder_internal_rotational_velo, shoulder_horizontal_abduction_fp, BMI

Step 1c ✅ — Strength velo model
  Files: pitchlens/analytics/velo_model.py (same file)
  - StrengthVeloModel: HistGradientBoostingRegressor
  - 20 features: CMJ/IMTP/hop test/relative strength/ROM + playing_level_enc
  - Filtered to pitch_speed_mph >= 60 (removes non-pitcher assessments and youth outliers)
  - KFold CV (5-fold, shuffled)
  - CV R²=0.342  RMSE=5.19 mph
  - Ridge baseline R²=-51 (confirmed linear model is inappropriate for this population spread)
  - Top features: peak_power CMJ, hop test RSI, shoulder IR ROM, jump height, body weight
  - LaunchpadDiagnostic dataclass combines both models + primary_recommendation

Step 1d ✅ — Composite mechanic scores
  Files: pitchlens/analytics/scoring.py
  - MechanicsScorer: empirical CDF from 411 OBP pitches → 0–100 percentile per variable
  - 5 composite scores: Arm Action, Block, Posture, Rotation, Momentum
  - Score distributions well-calibrated (mean ~50, std 10–20 per category)
  - Injury flags: elbow_varus_moment (UCL), shoulder_ir_moment (rotator cuff)
  - top_improvements() returns lowest-percentile components for coaching focus
  - score_cohort() batch-scores entire DataFrame

Step 1e ✅ — Peer matching
  Files: pitchlens/analytics/peer_match.py
  - PeerMatcher: cosine similarity on 62 normalized kinematic + energy features
  - find_comps() → top-N PitcherComp objects with similarity, velo, level, key metrics
  - velo_range_for_mechanics() → min/max/mean/median velo of top-N comps
  - level_breakdown() → playing level distribution of comps
  - Self-similarity = 1.000 ✓, same-session similarity ~0.95 ✓

KNOWN LIMITATIONS (address later, not blockers)
  - Strength model R²=0.34 is lower than target — reflects genuine difficulty of
    predicting velo from strength alone without playing level stratification.
    Acceptable for a capacity ceiling estimate; would improve with larger dataset.
  - elbow_transfer_fp_br dominates bio model importance (0.58 permutation importance).
    Biomechanically valid but worth monitoring — may be a proxy for overall athleticism.
  - Ridge regression catastrophically fails on hp_obp (R²=-51) — confirmed as
    population heterogeneity issue, not a bug. XGBoost handles it correctly.

PHASE 3: Computer Vision  ← THESIS TARGET
──────────────────────────────────────────────

Step 3a — MediaPipe pose extraction
  File: pitchlens/cv/pose_pipeline.py
  - Build on OBP's existing CV code: openbiomechanics/computer_vision/
  - MediaPipe Pose → 33 landmarks per frame at 30fps
  - Detect key events: foot plant (FP), max external rotation (MER),
    ball release (BR), max internal rotation (MIR)
  - Output: per-event landmark arrays matching OBP coordinate conventions

Step 3b — POI metric extraction from 2D video
  - Derive 2D approximations of key POI variables from MediaPipe output:
      hip_shoulder_separation, arm_slot, lead_knee_extension,
      torso_tilt, stride_length, cog_velo_x (approximate)
  - Validate against ground-truth OBP mocap values on held-out sessions
  - Build calibration regression: 2D MediaPipe estimate → 3D POI equivalent
  - Target metrics: the top-10 most important features from the bio model

Step 3c — End-to-end phone pipeline
  - Input: side-view video of pitcher (phone or laptop camera)
  - Output: mechanics scores + expected velo gap + top improvements
  - Latency target: < 5 seconds on a 2022 MacBook
  - Deployment: Streamlit page with video upload or webcam input
